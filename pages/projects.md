Most of my public work lives on GitHub, but the repos do not all represent the same kind of work. Some are projects I build and maintain end to end. Others are open-source or community repos where I contribute fixes, docs, and site work.

This page is the short index for the work that best represents how I like to build.

## Projects I maintain

### [django-vditor](https://github.com/pi-dal/django-vditor)

A Django package that brings [Vditor](https://github.com/Vanessa219/vditor) into admin forms and content workflows. It started as a practical editor integration, then grew into a more production-ready package with widgets, upload support, documentation, and a public demo site.

- [Project homepage](https://django-vditor.pi-dal.com/)
- [Source code](https://github.com/pi-dal/django-vditor)

### [Abraxas](https://github.com/pi-dal/Abraxas)

A personal meta-agent and AI-native agent experiment that grows through conversation instead of hardcoded behavior. The core stays intentionally small, while skills, plugins, memory artifacts, and channel adapters carry most of the behavior.

- [Repository](https://github.com/pi-dal/Abraxas)

## Projects I contribute to

### [ArozOS](https://github.com/tobychui/arozos)

An open-source web desktop operating system for low-power devices. I contributed fixes and documentation updates there, and it remains one of the projects that taught me how to work inside an existing codebase instead of only shipping my own ideas from scratch.

- [Repository](https://github.com/tobychui/arozos)
- [My commits](https://github.com/tobychui/arozos/commits?author=pi-dal)
